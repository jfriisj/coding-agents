# Epic 1.1 Quality Attribute Utility Tree

## Utility Tree Root
Design and explain an event-driven microservice architecture for real-time speech-to-speech translation that can be defended against the thesis drivers of latency, scalability, robustness, reproducibility, and privacy-safe evidence handling.

## Sub-Question to Pattern Map

| Sub-Question | Primary Patterns and Decisions |
|--------------|--------------------------------|
| SQ1 | Event-driven pipeline, lightweight orchestrator, schema governance, claim-check/object storage, VAD ingress gatekeeper |
| SQ2 | Service decoupling through brokered events, independently scalable workers, warm-loaded model execution, artifact persistence off the hot path |
| SQ3 | Correlation and trace propagation, runbook-driven measurement, allowlisted telemetry, benchmark and quality-evaluation harnesses |
| SQ4 | Retry/backoff, controlled degradation, at-least-once recovery, restart resilience, explicit failure-mode observation |
| SQ5 | Aggregate-only quality evaluation, baseline comparison narrative, evidence-ledger activation and deferment discipline |
