## Test Strategy (Pre-Implementation)
The objective is to verify that the security remediation for Epic 1.3 has been correctly implemented across API contracts (OpenAPI/AsyncAPI), architectural diagrams (Mermaid), and infrastructure configuration (Docker Compose). The focus is on ensuring the removal of insecure "token-in-URL" patterns, valid syntax for all artifacts, and linguistic consistency (English) as mandated by the project standards.

### Testing Infrastructure Requirements
**Test Frameworks Needed**:
- OpenAPI/AsyncAPI Syntax Validators (spectral/cli or editor-integrated)
- Mermaid CLI (to verify diagram renderability)
- Docker Compose CLI (to verify config validity)

**Testing Libraries Needed**:
- `grep` / `ripgrep` for security pattern and language detection.

### Required Unit Tests
- **Test-01: OpenAPI Syntax Validity**: Verify `openapi.yaml` has no syntax errors using `get_errors`.
- **Test-02: AsyncAPI Syntax Validity**: Verify `asyncapi.yaml` has no syntax errors using `get_errors`.
- **Test-03: Docker Compose Validity**: Verify `docker-compose.yml` is valid using `docker compose config`.
- **Test-04: Mermaid Syntax Validity**: Verify `Interaktionsflow.mmd` and `System- og Dataflow.mmd` have no syntax errors using `get_errors`.

### Required Integration Tests
- **Test-11: Security Regression (Token-in-URL)**: Ensure no `?token=` parameters exist in API specs or Mermaid diagrams.
- **Test-12: Security Hardening (Auth Handshake)**: Verify that AsyncAPI specifies an `AUTH` control message handshake requirement.
- **Test-13: Infrastructure Hardening (Port Binding)**: Verify that `docker-compose.yml` binds externally reachable ports to `127.0.0.1`.
- **Test-14: Language Consistency (English)**: Scan all changed files for Danish keywords (e.g., "Lyd", "Vært", "Gæst") to ensure 100% English normalization.

### Acceptance Criteria
- [ ] 0 Syntax errors across all 5 primary artifacts.
- [ ] 0 Occurrences of insecure token transport in contracts/diagrams.
- [ ] 100% Loopback binding for published services in `docker-compose.yml`.
- [ ] 100% English language compliance for all public-facing API documentation.
