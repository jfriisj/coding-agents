## 3. Findings & Recommendations

### 3.1 Positive Security Practices
- **Shift-Left Documentation**: The plan explicitly requires documenting security trade-offs (e.g., the overhead of mTLS vs. the benefit of trusted service boundaries) in the thesis manuscript.
- **Traceability**: All security controls are anchored to the telemetry baseline (ADR 6), providing empirical evidence for the system's security posture.

### 3.2 Minor Recommendations (Informational)
- **Token Handling**: While mTLS is mentioned for the broker, the chapter should also briefly touch upon how the Ingress Gateway handles WebSocket authentication tokens (e.g., JWT validation) if persistent user sessions are part of the scope.
- **Ray Namespace Isolation**: Milestone 2 should ensure that the "Service Decomposition" narrative mentions namespace-level isolation for Ray Actors to prevent lateral movement if one model container is compromised.

- **Status**: APPROVED
