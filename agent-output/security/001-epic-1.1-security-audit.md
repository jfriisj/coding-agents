# Security Audit: Epic 1.1 Baseline Setup (Full 5-Phase)

**Review Date**: 2026-03-16
**Auditor**: Security Agent
**Scope**: Plan [agent-output/planning/001-epic-1.1-baseline-setup.md](agent-output/planning/001-epic-1.1-baseline-setup.md) and associated architectural baseline.

## 1. Executive Summary
The baseline plan for Epic 1.1 is security-aware but requires strengthening in **Evidence Handling** and **Telemetry Boundary Enforcement**. The shift towards a "Normal-vs-Debug" telemetry contract is a critical security control to prevent PII/PHI or sensitive system trace leakage into the final thesis manuscript. The plan is **APPROVED WITH CONTROLS**.

## 2. Phase 1: Architectural Security (Trust Boundaries)
The event-driven translation pipeline (ASR -> MT -> TTS) involves multiple trust boundaries:
- **Boundary 1**: Ingress/Orchestrator (External Input).
- **Boundary 2**: Broker (Kafka/Ray) - Sensitive Audio/Text data transit.
- **Boundary 3**: Model Services (ASR/MT/TTS) - Execution environment.

### 2.1 Findings
- **[A1-H] Unclear Data Sanitization in Normal Telemetry**: The plan defines "Normal Telemetry" (Success Criteria 2) but does not explicitly mandate a sanitization step for correlation IDs or metadata that might lead to user fingerprinting.
  - **Remediation**: Add a "Telemetry Sanitization Rule" to Milestone 3.2.

## 3. Phase 2: Code Security (Static Review)
*Note: This phase focuses on the "Plan" as a governing artifact.*

### 3.1 Findings
- **[C1-M] Ambiguous Secret Management in Evidence Inventory**: Milestone 3.1 ("Inventory Evidence Sources") mentions auditing `docs/`. Some documents (`iac-swarm-bootstrap.md`) contain tokens.
  - **Remediation**: The evidence inventory MUST include a "Sanitization Check" step to ensure no leaked secrets (tokens/keys) are part of the baseline.

## 4. Phase 3: Dependency & Supply Chain Security
The plan assumes existing benchmarks and docs are authoritative.
- **Finding**: No mention of verifying the integrity of cached model artifacts or benchmark scripts.
- **Control**: Milestone 1.1 must include a checksum verification for external evidence benchmarks.

## 5. Phase 4: Infrastructure & Config
- **Finding**: Use of Docker Swarm and Ray (mentioned in Roadmap) requires explicit "In-Transit Security" (TLS) for cross-node communication of audio/text events.
- **Control**: Boundary verification (Milestone 2.3) must include a "Transport Security Requirement".

## 6. Phase 5: Compliance (Privacy/GDPR)
- **Principle**: Data Minimization.
- **Concern**: "Debug Traces" are marked as non-shareable, but no control exists to prevent accidental inclusion if the `docs/` folder is bulk-referenced.

## 7. Security Verdict & Implementation Controls

| Severity | Finding | Control Requirement |
|----------|---------|---------------------|
| **HIGH** | Telemetry Leakage | Define an explicit "Privacy-Safe Normal Telemetry" schema. |
| **MEDIUM** | Evidence Pollution | Mandate a credential/secret scan of the `docs/` evidence set. |
| **MEDIUM** | Transport Security | Verify TLS termination at trust boundaries in Milestone 2.3. |

**Verdict**: `APPROVED_WITH_CONTROLS`
