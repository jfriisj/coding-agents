# Security Audit: Epic 1.2 Research Framing and Background (Full 5-Phase)

**Review Date**: 2026-03-17
**Auditor**: GitHub Copilot (Security Agent)
**Scope**: Plan [agent-output/planning/002-epic-1.2-research-framing-background-completion.md](agent-output/planning/002-epic-1.2-research-framing-background-completion.md) (v2) and architectural revision [agent-output/architecture/003-epic-1.2-revision-gap-analysis.md](agent-output/architecture/003-epic-1.2-revision-gap-analysis.md).

## 1. Executive Summary
The revised plan for Epic 1.2 is architecturally sound and correctly prioritizes research framing. However, the integration of **P99 < 2s** latency success criteria and the **uv-first test-runner** introduces new trust and privacy boundaries that must be hardened. The plan is **APPROVED WITH CONTROLS**.

## 2. Phase 1: Architectural Security (Trust Boundaries)
The "Success Criteria" (Introduction) and "Background Synthesis" (Background) chapters now explicitly reference **P99 end-to-end latency** and **uv-first reproducibility**.

### 2.1 Findings
- **[A1-H] P99 Latency & Timing Sanitization**: The requirement for "P99 < 2s" (Introduction) implies that the manuscript will contain high-resolution timing data. Individual timing samples (e.g., hop-to-hop duration) can sometimes leak system load or user-specific metadata.
  - **Control**: Milestone 2.2 MUST mandate that all performance claims rely on **aggregated P99/P95/P50 metrics** only, never raw individual request timestamps (sanitized by [docs/privacy-safe-logging.md](agent-output/Analysis-and-design-of-a-platform-for-real-time-speech-translation-/docs/privacy-safe-logging.md)).
- **[A2-M] Claim-Check Pattern (Background Synthesis)**: The Background chapter (Milestone 3.2) will describe the **Claim-Check pattern (MinIO)**. 
  - **Risk**: Describing this pattern without mentioning the **Security of Presigned URLs** (bearer credentials) or **Token Expiration** could lead a reader to implement an insecure version.
  - **Control**: Milestone 3.2 MUST ensure the background narrative includes "Short-lived Presigned URLs" as a security invariant of the Claim-Check pattern.

## 3. Phase 2: Code Security (Static Review)
This phase focuses on the `uv-first` test-runner and its impact on the thesis evidence.

### 3.1 Findings
- **[C1-H] Test-Runner Credential Leakage**: The `uv-first` logic (Section 9 of Plan 002) frequently interacts with environment variables (e.g., `BFF_GATEWAY_TOKEN_SECRET`). 
  - **Risk**: If reproducibility mapping (Milestone 4.1) includes raw command-line examples in the manuscript without sanitization, secrets will be leaked.
  - **Control**: Milestone 4.1 MUST ensure that the `002-epic-1.2-rq-chapter-traceability.md` and the manuscript using the `test-runner.md` use **placeholders** (e.g., `<SECRET>`) for all credentials.

## 4. Phase 3: Dependency & Supply Chain Security
The plan integrates `uv` for dependency management.
- **Finding**: Use of `uv pip install -r tests/requirements.txt` is excellent for reproducibility.
- **Requirement**: Milestone 5.1 MUST ensure that the final thesis appendix (C) includes a **Requirements Pinning Policy** to prevent supply-chain drift in future evaluations.

## 5. Phase 4: Infrastructure & Config
- **Finding**: Schema Governance (Avro) is now a core background topic (Revision v2).
- **Control**: The background chapter (Milestone 3.2) must explicitly mention **Compatibility Posture (BACKWARD)** to ensure that the system remains secure and stable during rolling upgrades of model-wrapping services.

## 6. Phase 5: Compliance (Privacy/GDPR)
- **Principle**: Privacy by Design.
- **Requirement**: Milestone 2.3 (Introduction: Scientific Rigor) MUST explicitly anchor the thesis in the **Privacy-Safe Observability Policy** ([docs/privacy-safe-logging.md](agent-output/Analysis-and-design-of-a-platform-for-real-time-speech-translation-/docs/privacy-safe-logging.md)) as a core Success Criterion for "Evidence Rigor".

## 7. Security Verdict & Implementation Controls

| Severity | Finding | Control Requirement |
|----------|---------|---------------------|
| **HIGH** | Latency Metadata Leak | Aggregate all timing data to P99/P95; no raw timestamps in manuscript. |
| **HIGH** | Credential Leakage | Mandate placeholder usage for `BFF_GATEWAY_TOKEN_SECRET` in all docs. |
| **MEDIUM** | Pattern Security | Explicitly narrate "Short-lived Presigned URLs" in Background synthesis. |
| **MEDIUM** | Schema Posture | Embed "Backward Compatibility" as a security/stability driver for EDA. |
